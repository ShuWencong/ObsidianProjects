# MeshFilter中mesh和sharedMesh变量的区别

### sharedMesh 和 mesh 的区别整理

`MeshFilter` 中的 `sharedMesh` 和 `mesh` 是两种用于引用网格数据的属性，它们的区别可以从以下几个方面整理清楚：

### 1. 是否共享

- `sharedMesh`**：**

- 是一个 **共享的网格数据**。

- 多个对象可以同时引用同一个 `sharedMesh`，它们共享相同的网格。

- 修改 `sharedMesh` 会影响所有引用它的对象。

- `mesh`**：**

- 是 `sharedMesh` 的 **独立副本**，用于当前对象。

- 只有当前对象会受到 `mesh` 修改的影响，其他对象的 `sharedMesh` 保持不变。

### 2. 修改影响范围

- `sharedMesh`**：**

- 修改后会影响所有引用该 `sharedMesh` 的对象，因为它们共享同一份数据。

- 通常用于静态或全局一致的网格。

- `mesh`**：**

- 修改后只影响当前对象，因为 `mesh` 是一个独立的副本。

- 常用于需要动态修改网格的场景（例如，实时调整形状）。

- 如果既设置了`sharedMesh`也设置了`mesh` ，优先使用`mesh` 。

### 3. 使用场景

- `sharedMesh`**：**

- 适合多个对象使用相同的网格数据，例如一片草地或相同的树木。

- 在 **编辑器模式** 和 **运行时** 都可以安全使用。

- 修改 `sharedMesh` 会改变资源本身，因此慎用。

- `mesh`**：**

- 适合需要对某个对象的网格进行 **动态修改** 的场景，例如在运行时改变顶点或形状。

- 通常在运行时使用，访问时会自动生成独立副本。

### 4. 性能与内存

- `sharedMesh`**：**

- 更高效，多个对象共享同一份内存数据。

- 不会频繁创建副本，适合静态数据。

- `mesh`**：**

- 每次访问都会生成一个独立的网格副本，可能增加内存使用和性能开销。

- 如果频繁修改，需注意性能影响。

### 5. 运行时行为

- `sharedMesh`**：**

- 不会自动生成副本。

- 修改时，其他共享对象的网格会立即更新。

- `mesh`**：**

- 如果之前未访问过 `mesh`，Unity 会在访问时基于 `sharedMesh` 自动创建一个副本。

- 修改后，只有当前对象使用新的副本。

### 实践建议

- **使用 **`sharedMesh`** 的场景：**

- 当多个对象使用相同的网格数据，并且无需动态修改。

- 提升性能和内存效率，避免不必要的网格副本生成。

- **使用 **`mesh`** 的场景：**

- 当需要动态修改网格数据，例如实时变形、动画效果或顶点操作。

- 确保其他对象的网格不会受到影响。

- **避免频繁访问 **`mesh`**：**

- 每次访问 `mesh` 都可能生成新的副本，影响性能。

- 如果需要多次修改网格，最好缓存到本地变量后再操作。
