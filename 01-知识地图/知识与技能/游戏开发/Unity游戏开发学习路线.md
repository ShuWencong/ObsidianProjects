## 阶段划分

### 第一阶段：熟悉 Unity 基本工作方式

- [[Unity引擎基础]]：场景、GameObject、Component、Prefab、生命周期、Inspector、脚本挂载
- [[UI系统]]：Canvas、Text、Button、Image、布局、事件响应

这一阶段的目标是：理解 Unity 项目的基本结构，能用现有组件搭出简单交互界面。

### 第二阶段：掌握核心引擎模块

- [[物理与碰撞]]：Rigidbody、Collider、Trigger、Raycast、碰撞层级
- [[动画系统]]：Animator、Animation Clip、状态机、参数切换、角色动画
- [[资源管理]]：Prefab、材质、贴图、音频、Addressables、加载与卸载

这一阶段的目标是：能把角色、场景、碰撞、动画、资源组织成一个可运行 Demo。

### 第三阶段：处理表现和发布

- [[渲染与图形]]：材质、Shader、光照、后处理、渲染管线、特效基础
- [[性能优化]]：Profiler、GC、Draw Call、对象池、内存、加载卡顿
- [[平台发布]]：Windows、Android、iOS、WebGL、包体、权限、平台差异

这一阶段的目标是：能优化 Unity 项目表现，并把项目发布到目标平台。

## 推荐学习顺序

1. 先补上层基础：[[CSharp与编程基础|C#与编程基础]]、[[数学基础]]、[[调试与问题定位]]
2. 再学 [[Unity引擎基础]] 和 [[UI系统]]
3. 做小 Demo 时补 [[物理与碰撞]]、[[动画系统]]、[[资源管理]]
4. Demo 变复杂后补 [[性能优化]] 和 [[渲染与图形]]
5. 准备交付时学习 [[平台发布]]

## 练习建议

- 每学一个 Unity 模块，都做一个最小可运行 Demo
- 每个 Demo 都记录：用到了哪个 Unity 系统、怎么配置、遇到什么问题
- 通用知识不要写在本页，回到 [[知识与技能]] 或对应主题页
- Unity 具体问题可以链接回 [[调试与问题定位]]

## 后续拆分方式

当某个模块内容变多时，拆成独立笔记：

- 概念类：解释“这是什么”
- 操作类：记录“怎么做”
- 问题类：记录“报错和解决方案”
- 项目类：记录“在 Demo 里怎么用”
