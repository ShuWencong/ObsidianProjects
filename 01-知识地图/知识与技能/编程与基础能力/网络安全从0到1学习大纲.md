# 网络安全从0到1学习大纲

这篇笔记记录从零开始学习网络安全的路线。主线是先打基础，再进入 Web 安全和合法靶场练习。

> [!warning] 合法边界
> 只测试自己的环境、公开靶场，或明确授权的系统。不要扫描、攻击、尝试登录或破坏别人的网站、学校网络、公司系统。

## 总路线

```text
环境准备
-> 计算机基础
-> Linux 基础
-> 网络基础
-> Web 基础
-> Python / Shell
-> 安全基础概念
-> Web 安全漏洞
-> 安全工具
-> 系统与主机安全
-> 靶场实战与报告
-> 选择方向深入
```

## 阶段 0：准备环境

目标：搭一个可以放心练习、不会误伤别人的学习环境。

- 安装 WSL Ubuntu 或虚拟机
- 学会使用 Windows Terminal、PowerShell、Linux 终端
- 准备浏览器、Burp Suite、VS Code
- 用 [[Obsidian]] 或当前 vault 记录学习笔记
- 选择合法靶场，例如 PortSwigger Web Security Academy、DVWA、WebGoat

## 阶段 1：计算机基础

目标：知道电脑和系统大概怎么工作。

- 文件、目录、路径
- 进程、端口、服务
- 用户、权限、管理员、root
- 环境变量
- 编码：ASCII、UTF-8、Base64、URL 编码
- 常见文件类型：txt、json、xml、html、log、exe、sh、py

## 阶段 2：Linux 基础

目标：能在 Linux 里移动、查看、修改和搜索文件。

重点命令：

```bash
pwd
ls
cd
cat
less
head
tail
cp
mv
rm
mkdir
grep
find
chmod
sudo
ps
ss
curl
wget
ssh
```

需要理解：

- `/home`
- `/etc`
- `/var/log`
- `/tmp`
- 权限：`rwx`
- 管道：`|`
- 重定向：`>`、`>>`
- Windows 磁盘在 WSL 中的位置：`/mnt/c`、`/mnt/d`

## 阶段 3：网络基础

目标：看懂一次网络请求发生了什么。

重点概念：

- IP 地址
- 子网、网关、NAT
- DNS
- TCP / UDP
- 端口
- HTTP / HTTPS
- 请求方法：GET、POST、PUT、DELETE
- 状态码：200、301、302、403、404、500
- Cookie、Session、Token
- TLS 和证书基础

常用命令：

```bash
ping
ipconfig
nslookup
tracert
curl
netstat
ss
```

相关笔记：[[网络]]

## 阶段 4：Web 基础

目标：能看懂一个网页和后端如何交互。

- HTML
- CSS 基础
- JavaScript 基础
- 浏览器开发者工具
- 表单提交
- URL 参数
- JSON
- REST API
- 登录流程
- 前端和后端的区别
- 数据库基础：表、字段、SQL 查询

## 阶段 5：编程基础

目标：能写小脚本辅助分析和自动化。

推荐先学 Python：

- 变量、条件、循环
- 函数
- 文件读写
- 正则表达式
- `requests` 库
- JSON 处理
- 命令行参数

再补一点 Shell：

```bash
for
if
grep
awk
sed
xargs
```

## 阶段 6：安全基础概念

目标：建立攻击面和防御思维。

- 漏洞是什么
- 威胁、风险、资产
- 身份认证 Authentication
- 权限控制 Authorization
- 加密、哈希、编码的区别
- 最小权限原则
- 输入校验
- 日志与审计
- 常见安全错误配置

## 阶段 7：Web 安全入门

目标：学习最常见的 Web 漏洞。

建议顺序：

1. SQL 注入
2. XSS 跨站脚本
3. CSRF
4. 文件上传漏洞
5. 命令注入
6. 目录遍历
7. 认证绕过
8. 访问控制漏洞
9. SSRF
10. XXE
11. 反序列化漏洞
12. 业务逻辑漏洞

推荐靶场：

- [PortSwigger Web Security Academy](https://portswigger.net/web-security)
- [TryHackMe](https://tryhackme.com/)
- [Hack The Box Academy](https://academy.hackthebox.com/)
- DVWA
- Pikachu
- WebGoat

## 阶段 8：常用安全工具

目标：知道工具能做什么，而不是只会点按钮。

- Burp Suite：抓包、改包、重放请求
- Nmap：端口扫描
- Wireshark：抓包分析
- curl：手动构造请求
- Gobuster / dirsearch：目录扫描
- sqlmap：SQL 注入辅助验证
- Metasploit：了解基本概念，不急着依赖

## 阶段 9：系统与主机安全

目标：理解服务器和电脑如何加固，以及如何排查异常。

- Linux 用户和权限
- SSH 安全
- 防火墙基础
- 日志分析
- 进程排查
- 定时任务
- Web 服务：Nginx / Apache
- 数据库安全
- Docker 基础
- Windows 用户、服务、事件日志

## 阶段 10：攻防实战与报告

目标：把知识串起来，形成完整测试和复盘流程。

合法测试流程：

```text
信息收集
-> 端口和服务识别
-> Web 指纹识别
-> 漏洞验证
-> 权限影响分析
-> 修复建议
-> 写报告
```

报告应包含：

- 漏洞名称
- 影响范围
- 复现步骤
- 风险等级
- 修复建议
- 验证方式

## 阶段 11：方向分支

入门之后可以选择一个方向深入：

- Web 渗透测试
- 安全运维 / 蓝队
- 应急响应
- 逆向工程
- 二进制安全
- 云安全
- 移动安全
- 代码审计
- 安全开发
- 等保 / 合规 / 风险评估
