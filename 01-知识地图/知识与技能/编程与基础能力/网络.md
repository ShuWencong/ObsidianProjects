# 网络

这个主题用来沉淀计算机网络相关知识，目标是把概念、协议、工具和排错经验整理成可以复用的能力。

## 这是什么

网络关注的是设备之间如何连接、传输数据、定位服务、处理错误和保障安全。

## 为什么重要

- 写后端、客户端、数据库、分布式系统都会遇到网络问题
- 很多线上故障本质上是连接、延迟、超时、DNS、TLS 或协议理解问题
- 网络知识能直接提升调试、排错和系统设计能力

## 我现在怎么理解

先不要急着背完整模型，而是抓住一条主线：

1. 数据从我的电脑出发
2. 经过网卡、局域网、路由器和公网
3. 找到目标服务器
4. 建立连接
5. 按协议交换数据
6. 出错时通过工具定位问题

## 学习路线

1. 网络基础：IP、端口、局域网、公网、网关、NAT
2. TCP/IP：三次握手、四次挥手、可靠传输、拥塞控制
3. DNS：域名解析、递归查询、缓存、常见解析问题
4. HTTP：请求响应、状态码、Header、Cookie、缓存
5. HTTPS：TLS、证书、加密、证书链
6. 常用工具：`ping`、`tracert`、`nslookup`、`curl`、Wireshark
7. 实战排错：超时、连接失败、DNS 失败、证书错误、接口异常

## 练习方式

- 用 `ping` 和 `tracert` 观察一次访问经过了哪些节点
- 用 `nslookup` 查询常见网站的域名解析结果
- 用 `curl -v` 查看一次 HTTP/HTTPS 请求的细节
- 用 Wireshark 抓一次 DNS 或 HTTP 请求
- 写一个最简单的客户端和服务端，观察端口、连接和数据传输
