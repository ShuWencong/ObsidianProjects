# 回溯算法

回溯算法能解决如下问题：

- 组合问题：N个数里面按一定规则找出k个数的集合

- 排列问题：N个数按一定规则全排列，有几种排列方式

- 切割问题：一个字符串按一定规则有几种切割方式

- 子集问题：一个N个数的集合里有多少符合条件的子集

- 棋盘问题：N皇后，解数独等等

# 一、组合问题

### 组合问题去重策略：

- **使用 startIndex**：在组合问题中，`startIndex` 用于确保我们在选择元素时不会回头选择之前已经考虑过的元素。

- **树枝不能重复选同一个索引**：在递归调用时，通过传递 `i + 1` 作为 `startIndex`，我们确保了每个元素在每个组合中只出现一次。

- **树枝不能选同一个值**：如果数集中有重复的值，并且我们希望在组合中不重复这些值，我们需要进行树枝去重。这通常通过一个布尔数组来实现，其中值作为键，是否被选取作为值。

- **数组中同一个元素可以无限取**：当数组中元素可以重复选取时，我们需要传递 `i` 而不是 `i + 1` 给下一层的 `startIndex`。

- **树层去重**：在树层上，我们需要避免在同一层选择相同的元素。这需要我们在循环中加入一个条件判断 `(i > startIndex && array[i] == array[i - 1])`。

# 二、排列问题

在排列问题中，去重策略与组合问题略有不同，因为排列问题允许元素重复，但是不能重复使用同一个位置。下面是排列问题的去重策略：

### 排列问题去重策略：

- **每层都从 0 到 Length-1**：在排列问题中，我们需要考虑之前未选择的元素，因为顺序不同，也是新排列。

- **树枝去重**：在排列问题中，树枝去重通常是指避免在同一组合中重复使用相同的元素。这通常通过使用一个布尔数组来实现，标记某个元素是否已经被使用过，`if(used[i]) continue;`。

- **树层去重**：在排列问题中，树层去重是指避免在同一递归层级中选择相同的元素。这需要我们在循环中加入一个条件判断 `if(i > 0 && nums[i] == nums[i - 1] && !used[i - 1]) continue;`，以确保不会在相同的层级上选择相同的元素。

### C# 示例代码：

以下是一个C#示例，它实现了排列问题的去重策略：

```csharp
using System;
using System.Collections.Generic;
public class Solution
{
    public IList> PermuteUnique(int[] nums)
    {
        List> result = new List>();
        Array.Sort(nums); // 先排序，方便后续去重
        bool[] used = new bool[nums.Length];
        backtrack(nums, new List(), result, used);
        return result;
    }
    private void backtrack(int[] nums, List path, List> result, bool[] used)
    {
        if (path.Count == nums.Length)
        {
            result.Add(new List(path));
            return;
        }
        for (int i = 0; i  0 && nums[i] == nums[i - 1] && !used[i - 1])
                continue;
            // 树枝去重，如果当前元素已被使用，则跳过
            if (used[i])
                continue;
            // 做选择
            used[i] = true;
            path.Add(nums[i]);
            // 递归
            backtrack(nums, path, result, used);
            // 撤销选择
            used[i] = false;
            path.RemoveAt(path.Count - 1);
        }
    }
    public static void Main(string[] args)
    {
        Solution solution = new Solution();
        var permutations = solution.PermuteUnique(new int[] { 1, 1, 2 });
        foreach (var permutation in permutations)
        {
            Console.WriteLine(string.Join(", ", permutation));
        }
    }
}
```

在这个示例中，`backtrack` 方法实现了递归回溯，用于生成排列。在进入递归之前，我们进行了树层去重的判断，确保同一层级的递归不会选择相同的元素。同时，我们使用了一个布尔数组 `used` 来标记某个元素是否已经被使用过，从而实现树枝去重。在 `Main` 方法中，我们调用 `PermuteUnique` 方法来生成一个数组中的所有不重复排列，并打印结果。

# 五、棋盘问题

### 棋盘问题去重策略：

- **通过 for 循环遍历：**对于解数独，是两个 for 循环遍历整个棋盘，因为每个棋盘都是有对应的值的，需要给每个位置找到对应的值。而对于 N 皇后而言，只需 for 循环遍历行，选择那个唯一的皇后。

- **递归自身或者递归下一行**：对于解数独而言，直接反复的递归这个方法，不断完善棋盘即可。而对于 N 皇后而言，则是一行一行不断往下递归，直到选完 N 行的皇后。
