# 常量

# 一、常量类型

const 编译时常量

readonly 运行时常量

static 静态

static readonly 运行时静态变量

eg：

public class Animal

{

public static string str1 = "Animal";//类成员

public const float Pi = 3.14f;//类成员

public readonly float Pi2 = 3.14f//对象成员

public static readonly Animal animal;//类成员

static Animal(){	}

}

# 二、编译时与运行时常量

运行时常量（Runtime Constants）和编译时常量（Compile-time Constants）是编程中用于描述不同时间点确定的常量值的术语。它们在代码执行过程中发挥着不同的作用。

### 编译时常量（Compile-time Constants）

编译时常量是在编译期确定的常量值，这意味着在代码编译时，这些常量的值已经被确定，并且在编译完成后不可更改。

#### 特点：

- **确定时间**：值在编译期间就已经确定。

- **性能优势**：编译时常量可以直接嵌入到编译生成的二进制代码中，避免了运行时的计算，因而通常有性能优势。

- **使用限制**：编译时常量通常只能用于基本数据类型（如 `int`, `float`, `char`）和编译器能够在编译时计算出的表达式。

- **语言特性**：大多数编程语言提供的 `const` 或 `final` 修饰符可以用于定义编译时常量。

#### 示例（C#）：

```csharp
const int DaysInWeek = 7;
const double Pi = 3.14159;
```

在这个例子中，`DaysInWeek` 和 `Pi` 都是编译时常量，它们的值在编译期间已经确定，并且在程序运行时不会改变。

### 运行时常量（Runtime Constants）

运行时常量是在程序运行期间确定的常量值。这些常量的值可能在运行时计算出来，通常在程序运行后初始化，但在初始化后它们的值不会再改变。

#### 特点：

- **确定时间**：值在运行时确定，即代码在实际运行时通过某些操作计算得出。

- **灵活性**：运行时常量可以基于运行时的条件来赋值，例如从配置文件读取的值、数据库查询结果等。

- **性能影响**：因为值是在运行时确定的，因此可能需要额外的计算或资源访问，可能会对性能产生影响。

- **使用场景**：通常用于需要在运行时才能确定的值，但一旦确定后需要保证值不再改变的场景。

#### 示例（C#）：

```csharp
readonly int MaxUsers;
readonly string ConfigPath;

public MyClass()
{
    MaxUsers = GetMaxUsersFromConfig();  // 在运行时从配置中读取
    ConfigPath = Environment.GetEnvironmentVariable("CONFIG_PATH");
}
```

在这个例子中，`MaxUsers` 和 `ConfigPath` 是运行时常量，因为它们的值是在类实例化时根据运行时的条件（如配置文件、环境变量等）确定的。

### 区别总结

- **确定时间**：

- 编译时常量：在编译期间确定值。

- 运行时常量：在程序运行时确定值。

- **使用场景**：

- 编译时常量：适用于在编译期间可以确定的、不依赖于运行时数据的常量。

- 运行时常量：适用于需要在运行时计算或获取的值，并且这些值在初始化后不会再改变。

- **性能**：

- 编译时常量通常性能更好，因为它们在编译时就已经确定并内联在代码中。

- 运行时常量需要在程序运行时进行计算或赋值，可能对性能有一定影响，但提供了更多的灵活性。

理解编译时常量和运行时常量的区别有助于在编写代码时更好地选择适合的常量类型，以平衡性能和灵活性。
