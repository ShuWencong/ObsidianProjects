# Unity的场景序列化

在 Unity 中，序列化与反序列化的流程与场景/预制体生命周期紧密相关。以下是完整的操作流程及反序列化的具体阶段：

### 一、序列化流程（保存阶段）

- **触发条件**

- 保存场景（`Ctrl+S`）

- 修改预制体后应用（Apply Prefab）

- 脚本重新编译后数据恢复

- **执行步骤**

- **步骤 1**：遍历场景中所有激活的 `GameObject` 及其组件

- **步骤 2**：提取可序列化字段（公共字段或标记 `[SerializeField]` 的私有字段）

- **步骤 3**：将数据转换为 YAML 格式（场景文件）或二进制格式（预制体）

- **步骤 4**：写入 `.unity` 场景文件或 `.prefab` 预制体文件

### 二、反序列化流程（加载阶段）

#### 场景加载/进入播放模式时的反序列化阶段

- **触发条件**

- 打开场景文件

- 进入播放模式（Play Mode）

- 实例化预制体（`Instantiate`）

- **执行步骤**

- **步骤 1**：读取场景文件或预制体中的序列化数据

- **步骤 2**：创建空的 `GameObject` 并挂载组件（根据 `m_Component` 字段）

- **步骤 3**：**反序列化关键阶段**：将 YAML/二进制数据填充到组件字段中

- 覆盖脚本中通过 `= new List()` 等代码初始化的默认值

- 若字段未在序列化数据中定义，保留代码初始值

- **步骤 4**：重建对象引用（如 `Transform` 父子关系、脚本间的 `public` 引用）

### 三、关键特性与注意事项

- **覆盖优先级**

- 反序列化数据 > 代码初始化值 > 默认值

- 例如：若场景中序列化了 `driveHumanEntityWaitingList_`，则代码中的 `new List()` 会被覆盖

- **性能影响点**

- 反序列化阶段（步骤 3）是场景加载的主要耗时环节，尤其对含大量组件或复杂数据结构的对象

- **特殊类型处理**

- 多态类型需使用 `[SerializeReference]`（Unity 2019.3+）

- 字典（`Dictionary`）需自定义序列化逻辑

### 四、验证反序列化结果的调试方法

- **Inspector 对比**

- 在编辑器中对比脚本字段值与 Inspector 显示值，确认反序列化是否成功

- **场景文件分析**

- 直接查看 `.unity` 文件中的 YAML 数据，定位未正确序列化的字段

- **生命周期日志**

```csharp
void Awake() {
    Debug.Log("反序列化后的值: " + driveHumanEntityWaitingList_.Count);
}
```

- 在 `Awake` 中输出字段值，验证反序列化是否在脚本初始化前完成
